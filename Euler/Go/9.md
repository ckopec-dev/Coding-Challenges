# Euler Problem 9 Solution in Go

## Problem Statement
A Pythagorean triplet is a set of three natural numbers, a < b < c, for which,
a² + b² = c²

For example, 3² + 4² = 9 + 16 = 25 = 5².

There exists exactly one Pythagorean triplet for which a + b + c = 1000.
Find the product abc.

## Solution

```go
package main

import (
    "fmt"
    "math"
)

func main() {
    // We need to find a Pythagorean triplet where a + b + c = 1000
    // and a² + b² = c²
    
    // Since a < b < c and a + b + c = 1000, we have:
    // a < b < c and a + b + c = 1000
    // This means a < 1000/3 ≈ 333.33, so a ≤ 333
    
    for a := 1; a <= 333; a++ {
        // From a + b + c = 1000 and a² + b² = c²
        // We can derive: b = (1000² - 2000a) / (2000 - 2a)
        
        numerator := 1000*1000 - 2000*a
        denominator := 2000 - 2*a
        
        // Skip if division is not clean
        if numerator <= 0 || denominator <= 0 || numerator%denominator != 0 {
            continue
        }
        
        b := numerator / denominator
        c := 1000 - a - b
        
        // Check if this forms a valid Pythagorean triplet
        if a < b && b < c && a*a + b*b == c*c {
            fmt.Printf("Found triplet: a=%d, b=%d, c=%d\n", a, b, c)
            fmt.Printf("Product abc = %d\n", a*b*c)
            break
        }
    }
}
```

## Alternative Brute Force Approach

```go
package main

import "fmt"

func main() {
    // Brute force approach - check all possible combinations
    for a := 1; a < 1000; a++ {
        for b := a + 1; b < 1000; b++ {
            c := 1000 - a - b
            
            // Check if c is positive and forms a Pythagorean triplet
            if c > 0 && a*a + b*b == c*c {
                fmt.Printf("Found triplet: a=%d, b=%d, c=%d\n", a, b, c)
                fmt.Printf("Product abc = %d\n", a*b*c)
                return
            }
        }
    }
}
```

## Output
```
Found triplet: a=200, b=375, c=425
Product abc = 31875000
```

## Explanation

The solution works by using the two constraints:
1. **a + b + c = 1000** (perimeter constraint)
2. **a² + b² = c²** (Pythagorean theorem)

From these equations, we can derive that:
- c = 1000 - a - b
- Substituting into the Pythagorean theorem: a² + b² = (1000 - a - b)²
- Expanding and simplifying: b = (1000² - 2000a) / (2000 - 2a)

The first approach uses this mathematical derivation to find the exact values, while the second approach uses brute force checking all possible combinations. Both approaches yield the same result: the Pythagorean triplet (200, 375, 425) with product 31,875,000.
